# OctoAcme Project Management Docs

Welcome to OctoAcme's project management process documentation. This repository contains our standardized approach to initiating, planning, executing, and delivering projects with clear outcomes, defined roles, and continuous improvement.

## OctoAcme Project Management Approach

OctoAcme runs projects using a lightweight, iterative approach focused on customer value, clear ownership, data-informed decisions, and psychological safety. Our approach spans seven key phases:

### 1. **Initiation**
Capture the problem statement, identify stakeholders, define success metrics, and create a one-pager. A decision gate determines whether to proceed to planning based on clarity of success metrics and stakeholder alignment.

### 2. **Planning**
Transform an approved initiative into an actionable plan. Create a prioritized backlog with acceptance criteria, estimate work, define the Definition of Done, identify dependencies, and establish a release plan.

### 3. **Execution & Tracking**
Execute day-to-day delivery with a regular team rhythm: daily standups (15 min), weekly delivery syncs, and sprint demos. Use a project board (e.g., GitHub Projects) to track progress through Backlog → Ready → In Progress → In Review → QA → Done.

### 4. **Risk Management & Communication**
Maintain a risk register to identify, assess, and mitigate risks. Communicate progress and blockers through weekly status updates, stakeholder briefings, and escalation paths (team-level → PM → Product Lead → Sponsor).

### 5. **Release & Deployment**
Standardize releases to production by following pre-release requirements (acceptance criteria met, CI passing, release notes drafted), deployment checklists, smoke tests, and rollback playbooks.

### 6. **Retrospectives & Continuous Improvement**
After each sprint, release, or milestone, hold a retrospective to capture learnings, identify improvements, and track action items. Celebrate wins and iterate based on evidence.

### 7. **Roles & Personas**
Clear accountability through defined roles: Product Managers (define outcomes), Project Managers (coordinate delivery), Developers (implement features), QA/Testing (validate quality), and Stakeholders (provide inputs and approvals).

## Quick Start

1. **Starting a new project?** Begin with the [Project Initiation Guide](octoacme-project-initiation.md) to create a one-pager and validate business needs.
2. **Ready to plan?** Use the [Project Planning](octoacme-project-planning.md) doc to break work into shippable increments and create your backlog.
3. **Executing?** Reference [Execution & Tracking](octoacme-execution-and-tracking.md) for team cadences and workflows.
4. **Preparing a release?** Check the [Release & Deployment Guide](octoacme-release-and-deployment.md) for pre-release requirements and deployment steps.
5. **Reflecting on outcomes?** Follow the [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) guide to capture learnings and iterate.

## Key Principles

- **Customer-first**: Prioritize customer value and usability.
- **Iterative delivery**: Deliver small, testable increments and gather feedback early.
- **Clear ownership**: Each project has named Product Manager and Project Manager roles.
- **Data-informed decisions**: Measure impact and iterate based on evidence.
- **Psychological safety**: Encourage feedback, learning, and continuous improvement.
